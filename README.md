# ViaVpn Telegram Bot

Этот проект — не полноценная коммерческая VPN-платформа, а скорее практичный Telegram-обвязочный сервис для управления VPN-доступом через панель 3x-ui. Он помогает автоматизировать базовые операции: создавать пользователей, выдавать ссылки, показывать QR-коды, вести поддержку и выполнять простые админ-задачи. Для больших сетей, высокой нагрузки и серьёзной production-эксплуатации он подходит только частично.

## Назначение проекта

Проект рассчитан на небольшие и средние сценарии использования. В реальности он может стабильно работать на уровне примерно 100–200 пользователей, но это зависит от:

- мощности VPS;
- количества одновременно активных соединений;
- качества настроек 3x-ui;
- нагрузки на API и Telegram-ботов.

Если у вас большой трафик, много пользователей и высокая конкуренция по соединениям, лучше рассматривать более полноценную инфраструктуру с мониторингом, отдельными узлами, балансировкой и более сложной архитектурой.

## Что умеет проект

- создавать VPN-пользователей из Telegram;
- выдавать ссылки для подключения;
- генерировать QR-коды;
- работать с базовыми инвайтами и рефералами;
- вести простую поддержку через отдельный бот;
- хранить данные локально в SQLite.

## Ограничения проекта

Важно понимать, что этот сервис не является полноценной production-системой в классическом смысле. У него есть ограничения:

- он не рассчитан на очень высокую нагрузку;
- у него нет сложной панели управления, аналитики, автоматического масштабирования и продвинутого мониторинга;
- базовые функции бота реализованы достаточно просто;
- данные хранятся локально в SQLite, что подходит для небольшого проекта, но не для больших enterprise-сценариев.

## Что нужно для работы

На VPS должны быть:

- Ubuntu / Debian / другой Linux-сервер;
- доступ по SSH;
- установленный Docker и Docker Compose;
- установленная панель 3x-ui;
- Telegram-боты и их токены.

## 1. Подключение к VPS

Подключитесь к серверу по SSH:

```bash
ssh root@YOUR_SERVER_IP
```

Если нет SSH-ключей, лучше сразу настроить их. Парольный вход менее безопасен.

### Обновите систему

```bash
apt update && apt upgrade -y
```

### Установите базовые пакеты

```bash
apt install -y curl wget git unzip sudo
```

## 2. Установка Docker и Docker Compose

### Docker

```bash
curl -fsSL https://get.docker.com | sh
systemctl enable docker
systemctl start docker
```

### Docker Compose

```bash
sudo curl -L "https://github.com/docker/compose/releases/download/v2.29.2/docker-compose-linux-x86_64" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
```

Проверьте:

```bash
docker --version
docker-compose --version
```

## 3. Установка 3x-ui на VPS

3x-ui — это панель управления, через которую вы будете создавать пользователей, настраивать протоколы, конфиги и смотреть статистику. Без неё этот бот не сможет управлять VPN-клиентами.

### Установка

```bash
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

После установки откройте панель в браузере по адресу сервера и порту, который был указан во время установки. Обычно это что-то вроде:

```text
https://YOUR_SERVER_IP:YOUR_PORT
```

Войдите в панель, создайте администратора, если это не было сделано автоматически, и откройте раздел API. Там получите API-ключ.

### Что нужно сохранить из 3x-ui

Для работы бота нужны:

- `XUI_URL` — URL панели, например `https://YOUR_SERVER_IP:YOUR_PORT`;
- `XUI_API_KEY` — API-ключ панели.

> В примерах используются шаблонные значения. Замените их на реальные данные перед запуском.

## 4. Создание подключений в 3x-ui

Это ключевой этап. Здесь вы настраиваете, как пользователи будут подключаться к VPN.

### Основные протоколы

В 3x-ui можно создавать входы и выбирать протоколы, такие как:

- VLESS;
- Trojan;
- Hysteria.

### Какой вариант выбрать

- VLESS — хороший универсальный вариант, широко используется и поддерживается почти везде;
- Trojan — часто выбирают для обхода блокировок и стабильной работы;
- Hysteria — хороший вариант для нестабильных сетей, но требует более точной настройки.

Если вы только начинаете, обычно проще стартовать с VLESS или Trojan. Hysteria лучше подходит для более специфических задач и требует большего понимания сети.

## 5. Настройки протоколов и транспорта

В 3x-ui вы будете видеть параметры, связанные с транспортом и способом обмена трафиком. Ниже — краткое и практическое объяснение.

### WebSocket (WS)

Используется для маскировки трафика под обычный HTTPS/HTTP. Хорош для обхода блокировок и выглядит более естественно в сетях, где фильтруют VPN-трафик.

### TCP

Простой и стабильный транспорт. Подходит для базовых сценариев, где нет необходимости в сложной маскировке.

### gRPC

Более современный вариант, который иногда даёт хорошую стабильность, но требует аккуратной настройки и может быть сложнее в эксплуатации.

### TLS

TLS нужен для защиты соединения. В большинстве случаев его стоит включать. Без TLS соединение будет менее защищённым и хуже восприниматься сетью.

### MTU

MTU — это размер пакета. Если выставить его слишком большим или слишком маленьким, могут возникать проблемы с подключением, пропаданием трафика или нестабильностью. Лучше начинать с умеренных значений и проверять их на практике.

### Размеры пакетов

В реальных сетях важны не только протоколы, но и размеры пакетов. Если сеть нестабильная или маршрутизатор ведёт себя необычно, может потребоваться уменьшить размер пакета для более стабильной работы.

## 6. Настройки сети и BBR

### BBR

BBR — это алгоритм управления перегрузками в Linux. Он может улучшить пропускную способность и стабильность сети, особенно на современных VPS. Если у вас Ubuntu и современное ядро, стоит включить BBR.

### Как включить BBR

```bash
echo "net.core.default_qdisc=fq" | sudo tee /etc/sysctl.d/99-bbr.conf
echo "net.ipv4.tcp_congestion_control=bbr" | sudo tee -a /etc/sysctl.d/99-bbr.conf
sudo sysctl --system
```

Проверьте:

```bash
sysctl net.ipv4.tcp_congestion_control
sysctl net.core.default_qdisc
```

Ожидаемый результат: `bbr` и `fq`.

## 7. Настройка домена для панели

Для панели 3x-ui лучше иметь домен, а не только IP. Это улучшает внешний вид, упрощает работу и даёт возможность настроить нормальный HTTPS.

### Зачем нужен домен

- чтобы открыть панель по понятному адресу, а не по IP;
- чтобы получить нормальный HTTPS-сертификат;
- чтобы подключение выглядело более надёжно и профессионально;
- чтобы проще работать с пользователями и TLS.

### Что нужно сделать

1. Купить или использовать уже имеющийся домен.
2. Создать DNS-запись `A` или `CNAME` на ваш VPS.
3. Подключить домен к панели 3x-ui.
4. Настроить HTTPS, если это требуется.

Например:

```text
panel.example.com -> YOUR_SERVER_IP
```

## 8. Настройка Telegram-ботов

Создайте двух ботов через BotFather.

### Основной бот

1. Откройте BotFather.
2. Создайте нового бота.
3. Получите токен.
4. Сохраните его в `qq.env` как `BOT_TOKEN`.

### Бот поддержки

1. Создайте ещё одного бота в BotFather.
2. Получите токен.
3. Сохраните его в `qq.env` как `SUPPORT_BOT_TOKEN`.

### Получение вашего Telegram ID

Отправьте любое сообщение этому боту:

```bash
tg://user?id=YOUR_ID
```

Либо используйте @userinfobot.

Полученный ID укажите в `ADMIN_ID` и `SUPPORT_ADMIN_ID`.

## 9. Подготовка конфигурации

Отредактируйте файл `qq.env`:

```env
BOT_TOKEN=YOUR_MAIN_BOT_TOKEN
ADMIN_ID=YOUR_ADMIN_TELEGRAM_ID

SUPPORT_BOT_TOKEN=YOUR_SUPPORT_BOT_TOKEN
SUPPORT_ADMIN_ID=YOUR_SUPPORT_ADMIN_ID

XUI_URL=https://YOUR_SERVER_IP:YOUR_PORT
XUI_API_KEY=YOUR_3X_UI_API_KEY
DOMAIN=https://YOUR_DOMAIN_OR_SERVER_IP
```

> Не публикуйте этот файл в GitHub. Для публичного репозитория используйте `.env` или храните секреты только на сервере.

## 10. Установка зависимостей и запуск

На сервере выполните:

```bash
apt install -y python3 python3-pip python3-venv
cd /root/ViaVpn
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### Запуск через Docker Compose

```bash
cd /root/ViaVpn
docker compose up -d --build
```

### Запуск основного бота

```bash
cd /root/ViaVpn
source .venv/bin/activate
python via.py
```

### Запуск бота поддержки

```bash
cd /root/ViaVpn
source .venv/bin/activate
python support.py
```
